## Соответствие теории

| Алгоритм / вход | Рост при 2× n | Теория |
|---|---|---|
| Insertion, случайные | 3.8–4.1× | 4× O(n²) ✓ |
| Heap, случайные | 2.1–2.2× | ~2.2× O(n log n) ✓ |
| Insertion, возрастание | 2.0–2.1× | 2× O(n) ✓ |
| Heap, убывание | 2.1–2.2× | ~2.2× O(n log n) ✓ |

## Выводы

```
Вход              n < ~500      n > ~500
────────────────────────────────────────
Случайные         Insertion     Heap   ← 200× разрыв при n=50 000
Возрастание       Insertion     Insertion  ← O(n) vs O(n log n)
Убывание          Heap          Heap   ← Insertion деградирует до O(n²)
Неизвестный       Insertion     Heap
```

**Insertion** выигрывает только на отсортированных / почти отсортированных данных и малых n.  
**Heap** гарантирует O(n log n) при любом входе.  
Гибрид (Insertion для подмассивов ≤ 16–32 + Heap/Merge для больших) — основа `std::sort`.
